# 站点效率审视报告（基于代码与文档）

## 范围与依据
- 基于 `src/pages/*`、`src/components/*` 与 `docs/01_PRD.md` 进行静态审视。
- 结论偏向“效率提升”，不涉及大规模架构重写或跨端扩展。

## 现状概览（已实现）
- 入口：Landing（新用户引导），老用户可直达今日矩阵。
- 核心：矩阵页（任务录入、待办池、四象限拖拽、完成、推迟、删除、快捷键）。
- 辅助：档案页（日期导航、历史列表、导入导出、软删除）；仪表盘（统计与热力图）；回收站（恢复与永久删除）。

## 新增建议（提升“少点击/少切换/少摩擦”）
1. **全局搜索 / 过滤**：支持按关键字、完成状态、象限过滤，减少长列表查找成本。
2. **批量操作**：多选任务后一键移动象限、删除、推迟到明天，适配“整理模式”。
3. **任务快速编辑**：双击标题/备注内联编辑，避免删除重建。
4. **重复任务/模板**：支持“每日/每周”模板，一键生成常规任务。
5. **最近访问日期**：在档案页顶部展示最近 3-5 天，减少翻找。
6. **迁移偏好记忆**：对“昨日未完成任务迁移”的选择提供“记住选择”，降低重复弹窗。
7. **快速创建后自动聚焦**：可选开关，提交后保持输入聚焦（已存在），同时可设置默认进入整理模式。

## 删除/收敛建议（减少页面与路径）
1. **仪表盘入口收敛**：将关键统计卡片放入档案页顶部，仪表盘可改为折叠区或二级入口。
2. **回收站入口下沉**：放入档案页“更多操作”区域，减少主导航占位。
3. **确认弹窗策略统一**：对“删除/推迟/清空”等操作提供统一设置页，默认减少弹窗频率。

## 修改建议（细节体验与效率）
1. **任务条宽度与拖拽一致性**：拖拽镜像宽度始终与原卡片一致（已在修复中，建议验证宽屏/窄屏）。
2. **键盘体系补全**：增加“批量选择/取消”与“删除快捷键”提示，强化整理效率。
3. **排序与分组清晰化**：象限内任务支持“按创建时间/完成状态”切换排序。
4. **导入导出健壮性**：导入时明确提示“覆盖/合并”，提供失败报告。
5. **视觉文案统一**：完善中文文案，避免乱码与中英混杂造成认知成本。

## 优先级建议（Impact / Effort）
- **P0（高收益低成本）**：搜索/过滤、迁移偏好记忆、内联编辑、文案统一。
- **P1（中收益中成本）**：批量操作、最近访问日期、导入导出提示。
- **P2（高成本可选）**：仪表盘收敛改版、模板/重复任务。

## 风险与注意事项
- 当前数据结构为本地存储，批量操作与模板需要清晰的数据更新策略。
- 若调整信息架构（Dashboard/Recycle Bin 入口），需同步用户认知与文案提示。

## 结论
建议优先落地“搜索/过滤 + 内联编辑 + 迁移偏好记忆”，它们对效率提升最直接，且不破坏现有数据结构。其他结构性调整可作为二期迭代。
